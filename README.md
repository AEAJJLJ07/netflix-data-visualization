# Анализ данных и классификация контента Netflix (Недели 1–3)

## Описание проекта
Данный проект посвящен комплексному разведочному анализу данных (EDA) стриминговой платформы Netflix, проведению математико-статистического анализа и построению модели машинного обучения для автоматического определения типа контента на основе текстового описания.

## Цель проекта
Проанализировать структуру медиатеки Netflix, выявить скрытые закономерности (популярные жанры, динамику выпусков, пропуски данных) и обучить математическую модель классификации текстовых признаков контента с оценкой её качества.

## Датасет
- **Название датасета:** Netflix Movies and TV Shows
- **Источник:** Kaggle
- **Количество строк:** 8807
- **Количество признаков:** 12
- **Основные признаки:** `type` (целевой), `title`, `director`, `cast`, `country`, `release_year`, `rating`, `duration`, `listed_in`, `description`.

## Исследовательские вопросы
1. Каково точное процентное соотношение фильмов и сериалов на платформе?
2. Какие столбцы содержат наибольшее количество упущенных/пропущенных значений и как это влияет на анализ?
3. Какие слова в описании (description) являются ключевыми маркерами для отделения фильма от сериала?

## Этапы выполнения
1. Загрузка данных.
2. Первичный анализ данных / EDA.
3. Очистка данных (анализ пропущенных значений).
4. Математико-статистический анализ (описательные статистики, boxplot).
5. Визуализация данных (построение распределений и топов).
6. Построение интерактивного дашборда (в процессе разработки).
7. Построение ML-модели (Алгоритм Random Forest Classifier).
8. Интерпретация результатов и оценка метрик.

## ML-модель
- **Тип задачи:** Бинарная классификация (Classification).
- **Использованная модель:** Random Forest Classifier.
- **Целевая переменная:** `type` (Movie / TV Show).
- **Использованные признаки:** `description` (векторизованный через TF-IDF).
- **Метрики качества:**
  * Accuracy: 69.64% (базовая точность константного предсказания при несбалансированном текстовом наборе).
  * Подробный Classification Report доступен в логах выполнения программы.

## Результаты
В ходе математико-статистического анализа определено распределение контента по годам (средний год выпуска — 2010, медиана — 2011). Выявлены левые выбросы контента прошлых лет с помощью Boxplot. Построена модель машинного обучения. На защиту выносится аналитический вывод о сильном дисбалансе классов и сложности разделения типов контента исключительно по синопсису без учета прямых жанровых тегов, что наглядно подтверждает построенная матрица ошибок.
